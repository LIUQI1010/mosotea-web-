# Moso Tea — Design Context

## Design Context

### Users

**Public site:** Individuals, couples, and small groups (up to 8) considering booking a tea ceremony experience. Visitors arrive in a browsing/discovery mindset — often referred by word of mouth or searching for unique NZ experiences. Many are part of the Chinese-speaking diaspora (zh-TW support) or culturally curious locals. The job to be done: understand what the experience offers, feel emotionally drawn to it, and commit to a booking.

**Admin dashboard:** A single operator (the business owner) managing day-to-day bookings and time slots. The job to be done: quickly review pending bookings, confirm or cancel them, and manage availability — all with minimal friction. The admin is using this in a personal/home workshop context, not a corporate setting.

### Brand Personality

**Three words:** Unhurried, Artisanal, Grounded

**Voice and tone:** Warm and considered. Never rushed or transactional. Copy reads like a knowledgeable friend sharing a ritual, not a booking engine. In Chinese, it should feel literary and refined — not tech-translated.

**Emotional goal:** Calm anticipation. Visitors should feel the weight of the world lift slightly as they browse. The admin should feel the same warmth — not like they're using a generic SaaS tool.

### Aesthetic Direction

**Visual tone:** Minimal and organic. East Asian spatial sensibility — generous negative space, deliberate typographic hierarchy, no visual noise. Warm rather than cold, natural rather than digital.

**Theme:** Light mode only. The brand palette is inherently warm and light — a dark mode would undermine the aesthetic.

**References:** The public site itself is the reference. The admin dashboard should feel like it belongs to the same family — not a bolt-on tool.

**Anti-references:** Avoid anything that reads as generic SaaS (blue/gray color schemes, dense data tables, corporate sans-serif everything). Avoid overly playful or illustrative styles. Avoid emoji as UI elements.

### Design Principles

1. **One brand, two contexts** — The admin dashboard should share the same visual language as the public site: tea-brown accents, cream/off-white backgrounds, Cormorant Garamond for headings, Inter for body. A user who has seen both should immediately recognize they are the same product.

2. **Warmth over utility** — Functional elements (stats cards, status badges, navigation items) should still carry brand warmth. Use tea-brown and cream tones instead of generic white/gray. Status colors (pending, confirmed, cancelled) should be chosen to harmonize with the palette rather than clash with it.

3. **Icons over emoji** — All navigation and UI icons should use proper SVG icons (Heroicons or equivalent, stroke style at 1.5px weight to match existing usage throughout the codebase). Emoji are never used as UI affordances.

4. **Hierarchy through type** — Use Cormorant Garamond for section headings and card titles in the admin. Reserve Inter for data, labels, and body text. This creates a natural reading hierarchy that echoes the public site.

5. **Calm information density** — Stats and lists should breathe. Use generous padding, subtle borders (border-border / #E8E0D8), and rounded corners (rounded-xl / rounded-2xl) consistently. Avoid cramming — the owner checks this dashboard at their own pace, not in a high-pressure ops center.
